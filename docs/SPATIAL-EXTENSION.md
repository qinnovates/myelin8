# Engram Spatial Extension — Memory Layer for Spot

## Overview

Engram was designed for AI context persistence — compressing, indexing, and encrypting conversation history across tiers. The same architecture applies to **spatial memory**: LiDAR geometry, route data, object positions, and indoor maps generated by Spot (LiDAR navigation app).

This document specifies how Engram's four-tier compression, semantic indexing, and post-quantum encryption extend to spatial data — and how federated peer-to-peer sharing of spatial memory works using NSP (Neural Sensory Protocol) over BLE.

---

## Why Engram for Spatial Data

Engram already solves the core problems:

| Problem | Engram Solution | Spatial Application |
|---------|----------------|---------------------|
| Data grows forever | Tiered compression (hot → frozen) | Routes compress over time, old geometry archived |
| Need fast lookup without decompressing | Semantic index + keyword + embedding | "Stairs near Oak Street" without decompressing the full route |
| Data must stay private | Post-quantum encryption (ML-KEM-768) | User's spatial memory encrypted at rest |
| Need to verify integrity | SHA-256 hash per artifact | Detect tampered geometry (peer sharing) |
| Cross-session persistence | Artifact registry | Spatial memory persists across app launches |

## Tier Mapping

| Engram Tier | Age | Compression | Spot Spatial Data |
|-------------|-----|-------------|-------------------|
| **Hot** | Current session | 1x (uncompressed) | Live detections, current frame LiDAR, active route recording |
| **Warm** | 1-7 days | zstd-3 (4-5x) | This week's routes, recent bench/bathroom locations, RoomPlan scans |
| **Cold** | 7-90 days | zstd-9 + boilerplate strip (8-12x) | Last month's routes (semantic summary only), stale object positions |
| **Frozen** | 90+ days | Parquet columnar + delta (20-50x) | Archived routes, historical geometry, community-contributed data |

### What moves between tiers

```
HOT: Raw LiDAR mesh anchors, per-frame detections, live depth maps
     → After 24 hours, compress geometry to semantic summary

WARM: Route segments (position + detected objects + hazards)
      RoomPlan room layouts
      Bench/bathroom confirmed locations
      → After 7 days, strip redundant waypoints, keep hazard markers

COLD: Compressed route summaries ("Home to Store: 12 min, 6 stairs, 2 crossings")
      Indoor layouts (walls + doors + stairs only, furniture dropped)
      → After 90 days, columnar archive

FROZEN: Parquet-encoded route archive
        Historical geometry for deviation detection
        → Queryable but not automatically loaded
```

## Semantic Indexing for Spatial Data

Engram's existing semantic index (keywords + embeddings + HNSW + LSH) works for spatial queries:

**Keyword index examples:**
```
"stairs" → [route-segment#47, hazard-marker#12, room-scan#3]
"bench"  → [bench-location#8, bench-location#15, route-segment#22]
"Oak Street" → [route-segment#47, route-segment#48]
```

**Embedding search:**
Query: "covered seating near the park entrance"
→ Vector similarity finds bench-location artifacts with "covered=true" near park route segments

**Spatial search (NEW — extends Engram):**
Query by proximity: "What's within 50 meters of position X?"
→ H3 hexagonal grid index (same as Spot's data architecture) enables O(1) spatial lookup

## Encryption for Spatial Data

Engram's encryption architecture applies directly:

| Layer | Encryption | Spatial Note |
|-------|-----------|--------------|
| Per-artifact DEK | AES-256-GCM | Each route segment, room scan, object position encrypted separately |
| KEK (wraps DEKs) | ML-KEM-768 + X25519 hybrid | Post-quantum safe — spatial data may persist for years |
| Rust sidecar | Keys never in Python/Swift | Same sidecar, called from Spot via FFI or subprocess |
| Keychain integration | Touch ID / Face ID unlock | User's spatial memory requires biometric to decrypt |

**What gets encrypted:**
- All route geometry (positions, detected objects)
- Object memory (where you put your keys)
- Room scans (your home layout)
- Peer-shared geometry (encrypted in transit via NSP, encrypted at rest via Engram)

**What stays unencrypted:**
- Semantic index (keywords, embeddings) — needed for search without decryption
- OSM public data (benches, bathrooms from OpenStreetMap)
- Hazard type summaries (not positions — "this route has stairs" but not where exactly)

## Federated Sharing via NSP

### The Exchange Protocol

When two Spot users are near each other, Engram handles the data layer:

```
User A's Engram                          User B's Engram
┌─────────────┐                          ┌─────────────┐
│ Route #47   │                          │             │
│ (warm tier) │                          │  (empty)    │
│ 6 stairs    │                          │             │
│ 1 bench     │                          │             │
│ construction│                          │             │
└──────┬──────┘                          └──────┬──────┘
       │                                        │
       ▼                                        │
  Compress to semantic summary                  │
  (strip raw geometry,                          │
   keep: hazard types + counts                  │
   + relative positions)                        │
       │                                        │
       ▼                                        │
  NSP wrap:                                     │
  - Sign with device key                        │
  - Integrity hash (SHA-256)                    │
  - Encrypt with QKC session key                │
  - Merkle proof (path to root)                 │
       │                                        │
       ▼                                        ▼
  ─── BLE Multipeer Connectivity ──────────────►
                                         NSP verify:
                                         - Check signature
                                         - Verify integrity hash
                                         - Decrypt with QKC key
                                         - Validate Merkle proof
                                                │
                                                ▼
                                         Register in Engram:
                                         - Artifact type: peer-geometry
                                         - Confidence: Cs = 0.3 (low)
                                         - Tier: warm (auto-compresses)
                                         - Keywords indexed
                                                │
                                                ▼
                                         User B's LiDAR confirms:
                                         - Cs promoted to 0.85
                                         - Artifact stays in Engram
                                         - Now trusted for future recall
```

### Merkle Tree for Spatial Memory

Each user's Engram spatial data forms a Merkle tree:

```
              Root Hash (rotates with QKC)
             /                            \
      Routes Subtree                 POIs Subtree
      /           \                 /            \
  Route#47    Route#48        Bench#8       Bathroom#3
  (6 stairs)  (flat, clear)   (covered)     (wheelchair)
```

**Rollup:** A 30-minute walk generating 200 geometry snapshots is compressed into one Engram artifact with one Merkle leaf. One hash, one signature, one proof path. 200x less overhead than signing each snapshot.

**Proof:** To share just the stairs from Route#47, User A sends:
- Stairs semantic summary
- Merkle proof: Hash(Route#47) → Hash(Routes Subtree) → Root
- User B verifies the stairs data is part of User A's authenticated spatial memory without seeing any other routes

## Engram CLI Extensions (Proposed)

```bash
# Register spatial data from Spot
engram scan --type spatial --source ~/spot-export/routes/

# Search spatial memory
engram search "stairs near the park"

# Compress old routes
engram run --spatial --age-threshold 7d

# Export for peer sharing (NSP-wrapped)
engram export --nsp --artifact route-47

# Import peer geometry
engram import --nsp --verify peer-geometry.nsp

# Spatial stats
engram stats --spatial
# Routes: 23 (14 warm, 7 cold, 2 frozen)
# POIs: 47 (benches: 12, bathrooms: 8, hazards: 27)
# Total compressed size: 2.1 MB
# Compression ratio: 18.4x
```
